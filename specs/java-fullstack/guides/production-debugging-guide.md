# 生产问题排查指南

## 第一原则

- 先止血，再定位，再修复，再复盘。
- 不在生产环境盲目重启或直接改数据。
- 所有临时操作必须记录原因、命令、操作者和结果。

## 排查顺序

1. 确认影响范围：用户、接口、实例、时间窗口。
2. 查看监控：错误率、延迟、CPU、内存、连接池、线程池。
3. 使用 traceId 关联日志、链路和请求。
4. 检查最近发布、配置变更和数据库迁移。
5. 缩小到具体层：前端、网关、Controller、Service、Mapper、数据库、外部依赖。

## Java 常见指标

- JVM 内存、GC 次数和停顿。
- 线程池活跃数、队列长度、拒绝次数。
- HikariCP 活跃连接、空闲连接、等待连接。
- HTTP 客户端超时和错误率。
- 慢 SQL 和锁等待。

## 修复后

- 增加回归测试。
- 增加日志、指标或告警。
- 把根因沉淀到 `big-question` 或相关指南。
