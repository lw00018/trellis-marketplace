# API 版本管理指南

## 兼容变更

- 新增可选请求字段。
- 新增响应字段。
- 新增错误码但不改变原有错误语义。
- 放宽校验规则。

## 破坏性变更

- 删除或重命名字段。
- 修改字段类型或语义。
- 修改枚举 code 或含义。
- 修改分页结构、认证方式或错误码语义。
- 收紧校验规则导致旧客户端失败。

## 版本策略

- 路径版本使用 `/api/v1`、`/api/v2`。
- 小范围内部 API 可以使用 Header 或契约版本，但必须可追踪。
- 破坏性变更必须提供迁移说明和兼容期。

## 发布流程

- 先新增新版本 API。
- 前端和调用方迁移到新版本。
- 监控旧版本调用量。
- 确认无调用后下线旧版本。
