# 前端类型安全规范

## API 类型

- API 请求和响应类型必须来自后端契约或生成文件。
- 禁止为了临时通过编译使用 `any`。
- 后端字段可为空时，前端类型必须体现 null 可能性。
- 枚举值必须与后端稳定 code 对齐。

## 表单类型

- 表单输入类型不等于 API 请求类型，需要显式转换。
- 金额、时间、枚举在提交前必须转换为后端期望格式。
- 可选字段必须明确空字符串、null 和 undefined 的语义。

## 类型变更

- API 字段重命名必须全局搜索调用方。
- 修改枚举语义必须同步后端、前端、文档和测试。
- 不得只修改前端类型而不确认后端契约。
