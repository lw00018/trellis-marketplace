# 状态管理规范

## 状态分类

| 类型 | 示例 | 推荐位置 |
| --- | --- | --- |
| 服务端状态 | 用户列表、订单详情 | 请求缓存工具 |
| 页面状态 | 弹窗、筛选条件、选中项 | 页面或 feature |
| 全局客户端状态 | 主题、语言、当前用户 | 全局 store / context |
| 表单状态 | 输入值、校验错误 | 表单库或局部状态 |

## 规则

- 服务端状态不要重复复制到全局 store。
- 全局状态必须少而稳定。
- 缓存失效策略必须和后端写操作对应。
- 权限、用户信息和租户信息必须有单一可信来源。

## 常见问题

- 页面刷新后状态丢失说明状态不该只存在内存。
- 多处状态不一致说明缺少单一来源。
- 写操作后列表不更新说明缓存失效策略缺失。
