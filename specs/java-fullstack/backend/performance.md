# 性能规范

## 数据库性能

- 列表接口必须分页。
- 高频查询必须有索引和执行计划验证。
- 避免 N+1 查询，必要时使用批量查询或 JOIN。
- 大批量处理必须分批执行，控制事务大小。
- 慢 SQL 必须记录查询条件、耗时和 traceId。

## 连接池

- HikariCP 最大连接数必须结合数据库容量和应用实例数计算。
- 禁止无界扩容连接池。
- 连接泄漏必须开启检测或通过监控发现。
- 线程池和连接池容量必须匹配，避免线程远多于连接导致排队。

## 缓存

- 缓存必须有明确 key、TTL、失效策略和一致性预期。
- 不缓存权限敏感或强一致要求的数据，除非设计了失效机制。
- 防止缓存穿透、击穿和雪崩。
- 缓存 key 禁止包含未归一化的大对象或敏感信息。

## 外部调用

- 所有外部调用必须设置超时。
- 重试必须有最大次数、退避策略和幂等保护。
- 批量外部调用必须控制并发数。
- 关键外部依赖必须有降级或熔断策略。

## 异步处理

- 线程池必须显式配置队列容量和拒绝策略。
- 异步任务必须传递 traceId 和必要安全上下文。
- 异步失败必须可观测，不能只在后台静默失败。
- 定时任务必须防重复执行，分布式部署下使用锁或调度平台。
