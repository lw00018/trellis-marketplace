# 后端质量检查清单

## 提交前命令

```bash
mvn -B clean verify
mvn -B dependency:tree
mvn -B enforcer:enforce
```

## 代码质量

- [ ] 类、方法、变量命名符合阿里 Java 规范和团队约定。
- [ ] 没有无意义的 `Manager`、`Processor`、`Util` 泛化命名。
- [ ] 没有重复代码和过长方法。
- [ ] 没有裸泛型、魔法值、无意义注释。
- [ ] 公共 API 和复杂业务规则有必要文档。

## API 质量

- [ ] Controller 不包含业务规则。
- [ ] 请求 DTO 和响应 VO 分离。
- [ ] 参数校验完整。
- [ ] 错误码稳定且可定位。
- [ ] API 兼容性已评估。

## 数据库质量

- [ ] 更新和删除都有明确条件。
- [ ] 查询已分页或有明确数量上限。
- [ ] 排序字段来自白名单。
- [ ] SQL 参数绑定安全。
- [ ] 数据库结构变更有迁移脚本。

## 事务质量

- [ ] 事务边界位于 Service 层。
- [ ] 不存在自调用导致事务失效。
- [ ] 回滚规则明确。
- [ ] 事务内没有不可控远程调用。

## 安全质量

- [ ] 没有提交密钥和敏感配置。
- [ ] 日志不包含敏感信息。
- [ ] 权限校验覆盖资源归属。
- [ ] 外部输入全部校验。

## 测试质量

- [ ] 核心业务规则有单元测试。
- [ ] 数据库 SQL 有集成测试或可验证用例。
- [ ] 权限、异常、边界路径有覆盖。
- [ ] 测试不依赖执行顺序。
