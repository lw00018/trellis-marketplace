# 配置规范

## 配置来源

- 本地默认配置可以放在 `application-local.yml`。
- 测试和生产配置来自环境变量、配置中心或部署系统。
- 密钥、密码、Token、证书不得提交到仓库。
- 示例配置使用 `application-example.yml`，只包含非敏感占位值。

## 配置绑定

- 业务配置使用 `@ConfigurationProperties`。
- 配置类必须有前缀，例如 `app.payment`。
- 关键配置必须使用 Bean Validation 校验。
- 禁止在大量业务类中散落 `@Value`。

## 命名

- 配置 key 使用小写 kebab-case。
- 同一领域配置放在同一前缀下。
- 布尔配置使用肯定语义，例如 `enabled`，避免双重否定。

## 环境隔离

- 不同环境必须使用不同数据库、缓存、消息队列和第三方凭证。
- 生产配置不得依赖本地默认值。
- 测试环境不得连接生产资源。
- 删除或修改配置前必须搜索全部引用。

## 超时与容量

以下配置必须显式声明：

- HTTP 客户端连接超时、读取超时、总超时
- 数据库连接池最大连接数、最小空闲连接、连接超时
- 线程池核心线程数、最大线程数、队列容量、拒绝策略
- 缓存过期时间和最大容量
- 重试次数、退避策略和熔断阈值
