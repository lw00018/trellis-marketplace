# 测试规范

## 测试分层

| 类型 | 目标 |
| --- | --- |
| 单元测试 | 验证单个类或函数的业务规则 |
| 切片测试 | 验证 Controller、Mapper 等局部 Spring 能力 |
| 集成测试 | 验证数据库、事务、外部依赖集成 |
| 端到端测试 | 验证核心用户流程 |

## 单元测试

- Service 业务规则必须优先用单元测试覆盖。
- 单元测试覆盖率必须大于 85%。
- 测试名称必须描述场景和期望结果。
- Mock 只用于外部依赖，不要 Mock 被测对象内部逻辑。
- 禁止只覆盖 happy path，必须覆盖边界和异常路径。

## Spring 测试

- Controller 使用 `@WebMvcTest` 或集成测试验证参数、鉴权和响应结构。
- Mapper 使用真实数据库或 Testcontainers 验证 SQL。
- 需要完整上下文时才使用 `@SpringBootTest`。
- 测试 Profile 必须隔离外部资源。

## 数据库测试

- 数据库相关逻辑优先使用 Testcontainers 或与生产兼容的测试数据库。
- 每个测试必须独立，不能依赖执行顺序。
- 测试数据必须清晰，避免共享全局脏数据。
- 数据库迁移脚本必须在测试中执行。

## Maven 命令

```bash
mvn -B test
mvn -B verify
```

## 覆盖重点

- 权限边界。
- 事务回滚。
- 幂等行为。
- 分页和排序。
- 错误码和异常响应。
- 数据库唯一约束和并发更新。
