# 时区与 Instant

## 现象

- 前端显示时间比数据库早或晚 8 小时。
- 时间戳秒和毫秒混用导致日期异常。
- 跨时区用户看到错误日期。

## 根因

应用默认时区、数据库时区、JDBC 时区、前端展示时区不一致，或字段没有明确单位。

## 解决方案

- 服务端内部使用 `Instant` 表达时间点。
- 展示时结合用户 `ZoneId` 格式化。
- API 明确使用 ISO-8601 或 Unix 毫秒。
- 数据库、JVM、容器时区配置必须明确。

## 预防

- 字段命名和文档说明单位。
- 禁止手写固定天数换算月份和年份。
- 时间转换逻辑必须有测试覆盖。
