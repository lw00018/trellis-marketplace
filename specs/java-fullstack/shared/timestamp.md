# 时间与时区规范

## 基本原则

- 新代码使用 `java.time`。
- 禁止新业务使用 `Date`、`Calendar`、`SimpleDateFormat`。
- 服务端存储时间优先使用 UTC。
- 展示时间在用户时区格式化。

## 跨系统传输

- REST API 时间字段使用 ISO-8601 字符串或 Unix 毫秒。
- 必须明确字段单位，禁止前后端猜测秒或毫秒。
- 字段命名应体现语义，例如 `createdAt`、`expiresAt`。

## 数据库

- 审计时间字段统一为 `created_date`、`updated_date`。
- 数据库时区、JDBC 时区和应用默认时区必须明确配置。
- 禁止用本地时区时间做跨区域业务判断。

## 常见风险

- 夏令时导致一天不一定是 24 小时。
- 月份和年份不能用固定天数换算。
- 前端时间戳单位混用会导致时间偏移几十年。
