# Spring 事务自调用失效

## 现象

- 方法标注了 `@Transactional`，但异常后数据没有回滚。
- 同一个类中 A 方法调用 B 方法，B 方法的事务传播行为没有生效。

## 根因

Spring 声明式事务通常基于代理实现。只有通过代理对象调用的方法才会被事务拦截，同类内部 `this.method()` 调用不会经过代理。

## 解决方案

- 把事务方法移动到另一个 Service。
- 让外层公共方法承担完整事务边界。
- 避免把私有方法标注为 `@Transactional`。
- 必要时通过代理对象调用，但优先重构职责。

## 预防

- 事务统一放在 Service 公共方法上。
- Code Review 检查同类内部调用事务方法。
- 为关键事务写集成测试验证回滚。
