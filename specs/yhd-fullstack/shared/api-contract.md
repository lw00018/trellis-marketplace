# API 契约规范

## 契约内容

- 路径、方法、请求参数、响应结构、错误码、认证方式都属于 API 契约。
- 字段含义、枚举 code、时间格式、分页结构必须明确。
- 公开 API 必须有版本号。

## 变更规则

- 新增可选字段通常兼容。
- 删除字段、改名字段、修改字段语义、修改枚举语义属于破坏性变更。
- 破坏性变更必须升级版本或提供兼容期。
- 契约变更必须同步前端类型、后端 DTO、测试和文档。

## 错误码

- 错误码是契约的一部分，不能随意改名。
- 前端必须基于错误码而不是错误文案判断逻辑。
- 错误文案可以国际化，错误码必须稳定。

## 校验

- API 契约变更前先搜索所有调用方。
- 集成测试必须覆盖关键 API 的成功、失败和权限路径。
