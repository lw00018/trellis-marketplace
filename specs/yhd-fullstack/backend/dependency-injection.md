# 依赖注入规范

## 构造器注入

- 优先使用构造器注入。
- 禁止字段注入。
- 必需依赖使用 `final` 字段表达不可变性。
- 依赖过多说明类职责过大，应拆分职责。

## Bean 命名

- Bean 名称必须表达业务语义。
- 多实现不能只依赖默认类名猜测注入结果。
- 存在多实现时使用明确的 `@Qualifier`、策略 Map 或工厂。

## 条件装配

- 环境差异优先使用配置和条件装配。
- 禁止在业务代码中到处判断当前 Profile。
- 自动配置必须有开关和默认值说明。

## 循环依赖

- 循环依赖通常说明职责设计有问题。
- 禁止通过开启循环依赖配置掩盖设计问题。
- 通过抽取领域服务、事件、接口或重组模块解除循环。

## 生命周期

- 初始化逻辑使用 `@PostConstruct` 时必须快速、幂等且无远程重操作。
- 资源释放使用 `@PreDestroy` 或生命周期接口。
- 复杂启动任务使用 Runner，并明确失败策略。
