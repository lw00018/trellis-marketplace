# 身份认证与授权

## 认证模型

- Web 管理端优先使用 Session 或短期访问 Token + 刷新 Token。
- 开放 API 使用签名、OAuth2、JWT 或网关鉴权，必须明确过期和吊销策略。
- 禁止把长期有效 Token 存在前端不安全位置。
- 登录失败必须有频率限制和审计记录。

## JWT

- JWT 必须设置过期时间、签发方、受众和唯一标识。
- 签名密钥必须来自密钥系统或环境变量。
- JWT 只放最小必要声明，不存放敏感信息。
- 需要即时失效能力时必须配合黑名单、版本号或服务端会话状态。

## 权限模型

- RBAC 适合角色权限稳定的系统。
- ABAC 适合资源属性、组织、租户、状态共同影响权限的系统。
- 权限检查必须在服务端执行，前端隐藏按钮不等于授权。
- 资源级权限必须校验资源归属，不能只校验角色。

## Spring Security

- 安全配置必须集中管理，避免多个配置类相互覆盖。
- 鉴权失败返回 401，权限不足返回 403。
- CORS、CSRF、Cookie 策略必须结合部署形态设计。
- 管理端、开放 API、内部 API 应使用不同安全链或明确路径隔离。

## 审计

- 登录、登出、权限变更、敏感数据访问、关键业务操作必须记录审计日志。
- 审计日志必须包含操作人、资源、动作、结果、时间、traceId。
- 审计日志不得记录明文密码、完整 Token 或完整隐私数据。
