# 实现前检查清单

## 需求边界

- [ ] 需求输入、输出、异常路径已经明确。
- [ ] 已确认是否影响现有 API 契约。
- [ ] 已确认是否需要权限、审计、幂等和限流。
- [ ] 已确认是否涉及数据库结构或字段语义变更。

## 复用搜索

- [ ] 搜索是否已有相同领域模块。
- [ ] 搜索是否已有相同错误码、枚举、配置项。
- [ ] 搜索是否已有类似 Service、Mapper、DTO、VO。
- [ ] 搜索是否已有通用工具，避免重复实现。

## 技术方案

- [ ] Controller、Service、Mapper 职责划分清晰。
- [ ] 事务边界和回滚规则明确。
- [ ] 数据库查询有分页、索引和条件限制。
- [ ] 外部调用有超时、重试和降级策略。

## 测试计划

- [ ] 单元测试覆盖核心业务规则。
- [ ] 集成测试覆盖数据库和事务。
- [ ] 权限、异常、边界路径有用例。
- [ ] API 契约变更有前后端联调验证。
