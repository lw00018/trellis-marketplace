# 缺陷根因复盘指南

本指南用于修复非平凡前端缺陷后复盘根因，判断是否需要补充测试、规则或公共能力，避免同类问题再次出现。

## 何时使用

出现以下情况时，应按本指南复盘：

- bug 调试时间超过预期。
- 修复涉及页面、组件、API、Store、路由或缓存多处代码。
- 问题和框架、UI 库、Portal CMS、自动导入或 request 行为有关。
- 同类问题在多个页面重复出现。
- 修复方式看起来像绕过，而不是消除根因。

## 根因分类

### 1. 接口与数据契约

- [ ] API 入参或响应字段是否与 `src/model/` 类型不一致？
- [ ] 是否遗漏分页、排序、筛选、上传 URL 或 loading 配置？
- [ ] 是否在页面里临时处理了通用错误，而不是复用 request 统一能力？
- [ ] 是否误解了空值、默认值、状态码或枚举含义？

需要修复时，优先同步 API 封装、model 类型、表单默认值、表格展示和测试。

### 2. 路由、权限与 Portal CMS

- [ ] 是否从 `vue-router` 导入了 hooks 或类型？
- [ ] 是否绕过 Portal CMS 读取 token、用户或权限？
- [ ] 路由 meta、菜单显示、按钮权限或标签页标题是否遗漏？
- [ ] 刷新、重定向、关闭标签页是否没有使用 `useRedoPage`、`useGo` 或 `useTabs`？

这类问题不要通过手写全局 router 或改入口文件解决。

### 3. 状态与缓存

- [ ] 状态应该在页面局部、组件私有 composable、Pinia Store 还是缓存中？
- [ ] 是否直接使用了 `localStorage` / `sessionStorage`？
- [ ] Store 初始化、退出登录、标签页关闭、页面刷新时是否清理或恢复状态？
- [ ] keepAlive 页面是否出现旧数据、重复请求或未刷新问题？

缓存相关问题应统一使用 `localCache` / `sessionCache`，不要在页面里各自实现。

### 4. 组件生命周期与异步

- [ ] 请求返回时组件是否已经卸载？
- [ ] 弹窗关闭后是否残留表单状态？
- [ ] 多次点击是否导致重复提交或响应覆盖？
- [ ] watch、副作用、事件监听是否没有正确清理？
- [ ] 私有 composable 是否承担了不属于响应式流程的职责？

修复时优先收敛状态边界，而不是堆额外标志位。

### 5. UI 库与样式

- [ ] Ant Design Vue 或 YUI 组件是否有默认行为被误解？
- [ ] 全局注册组件是否遗漏，导致局部重复注册或运行异常？
- [ ] scoped Less、Tailwind、全局变量或 z-index 是否冲突？
- [ ] message、Modal、loading 是否重复封装或样式不一致？

使用 UI 库前应确认组件 API、默认值、事件名和插槽行为。

### 6. 自动导入与类型

- [ ] 是否手动编辑了 `types/auto-imports.d.ts` 或 `types/components.d.ts`？
- [ ] 自动导入来源是否和项目配置一致？
- [ ] 未使用变量、参数、类型错误是否被临时绕过？
- [ ] 类型是否重复定义，导致页面和 API 契约漂移？

不要通过降低 TypeScript 或 ESLint 严格度解决问题。

## 修复后检查

- [ ] 是否找到了真正根因，而不是只修了表面症状？
- [ ] 是否存在同类页面或组件也需要同步修复？
- [ ] 是否应该抽取到 `utils`、`hooks`、Store 或组件私有 composable？
- [ ] 是否需要补 Jest 单元测试或组件测试？
- [ ] 是否需要更新本规范中的规则或指南？

## 复盘记录建议

只在问题具有普遍性时补充规范；不要把单个业务事故、客户数据或临时上下文写进模板。可记录为：

```text
问题：某类字段/状态/缓存/路由行为容易误用。
根因：项目约定或第三方默认行为不明显。
防线：新增规则、测试或公共封装。
```
