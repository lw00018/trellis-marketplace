# 跨层变更思考指南

本指南用于处理同时影响多个前端层级的需求，避免只修改页面或接口的一处代码，导致类型、状态、路由、权限、文案或缓存不一致。

## 适用场景

当需求触及以下 3 类及以上内容时，应先按本指南检查：

- 页面或业务组件
- 私有子组件、`composable/`、`interface.ts`、`config.ts`
- `src/api/` 接口封装
- `src/model/` 类型、接口入参或响应结构
- Pinia Store
- Router、路由参数、菜单权限、标签页
- 表单默认值、校验规则、表格列、筛选项
- i18n 文案、消息提示、按钮文案
- 缓存、keepAlive、页面刷新、标签页关闭行为
- Ant Design Vue、YUI 或 Portal CMS 能力

## 前端 Vue3 工程常见层级

```text
用户操作 / 路由入口
        ↓
页面 Page
        ↓
业务组件 / 私有子组件
        ↓
composable / hooks
        ↓
Pinia Store / route query / 缓存
        ↓
API request
        ↓
Model 类型 / 接口契约
        ↓
Portal CMS / 权限 / token / 环境配置
```

每一层都可能有独立的默认值、类型、状态和异常处理。跨层需求必须检查链路是否一致。

## 变更前检查

### 1. 数据来源

- [ ] 数据来自接口、路由参数、Store、缓存还是组件 props？
- [ ] 是否已有 `src/api/` 封装，不需要在页面里直接写 request？
- [ ] 类型是否已经存在于 `src/model/` 或组件目录 `interface.ts`？
- [ ] 是否需要通过 Portal CMS 获取用户、token、权限或路由能力？

### 2. 数据流向

- [ ] 页面是否只展示数据，还是会写回接口？
- [ ] 表单提交后是否需要刷新列表、关闭弹窗、更新标签页标题或跳转路由？
- [ ] Store 是否需要同步更新，还是只依赖接口重新查询？
- [ ] 是否有 keepAlive 页面，需要处理刷新或缓存失效？

### 3. 类型与默认值

- [ ] API 入参与响应类型是否同步更新？
- [ ] 表单初始值、重置值、编辑回填值是否一致？
- [ ] 枚举、状态映射、静态选项是否放在组件 `config.ts` 或合适的共享位置？
- [ ] 可空字段在表单、表格、详情页和提交接口中是否解释一致？

### 4. UI 与交互

- [ ] 表格列、筛选项、详情展示、导出字段是否同步更新？
- [ ] 按钮权限、禁用态、loading、空状态、错误提示是否一致？
- [ ] 使用 Ant Design Vue 或 YUI 时是否复用项目已有封装？
- [ ] 是否需要 i18n；如果启用，用户可见文案是否全部接入？

### 5. 路由与权限

- [ ] 路由类型、`useRoute`、`useRouter` 是否来自 `@yhdfe-plugins/portal-cms-vue`？
- [ ] 新增路由是否放在 `src/router/modules/`，而不是改 `src/router/index.ts`？
- [ ] 路由 meta、菜单展示、权限展示是否和业务入口一致？
- [ ] 标签页标题、关闭、刷新行为是否需要使用 `useTabs` 或 `useRedoPage`？

### 6. 请求与异常

- [ ] 是否通过 `@/request` 发请求，而不是创建新的 HTTP client？
- [ ] 是否需要 `{ loading: true }`，避免页面重复手动控制进度条？
- [ ] 接口错误是否沿用全局 request 处理，不在页面重复封装通用错误逻辑？
- [ ] 上传、下载、分页、排序参数是否与现有 API 约定一致？

## 常见遗漏

| 变更 | 容易遗漏 |
| --- | --- |
| 新增接口字段 | `model` 类型、表格列、表单默认值、详情展示、测试 |
| 修改枚举含义 | `config.ts` 映射、筛选项、状态颜色、权限判断、文案 |
| 新增页面入口 | 路由模块、菜单权限、页面标题、标签页行为 |
| 新增缓存状态 | `localCache` / `sessionCache` 封装、Store 初始化、清理时机 |
| 新增公共交互 | 是否已有 Hook、是否应沉到 `src/hooks/` 或组件私有 `composable/` |

## 完成前确认

- [ ] 已搜索类似页面、接口、Store、Hook 或组件实现。
- [ ] 已同步更新类型、默认值、UI 映射和测试。
- [ ] 未修改 `src/main.ts`、`src/App.vue`、`src/plugins/index.ts`、`src/plugins/custom.ts` 等受保护文件来适配单个业务需求。
- [ ] 未绕过 Portal CMS、项目 request、缓存封装或路由封装。
- [ ] 用户可见文案、错误提示和空状态在同一功能内风格一致。
