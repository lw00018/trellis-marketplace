# 跨层思考指南

## 何时使用

- 功能同时涉及前端、Controller、Service、Mapper、数据库。
- 字段在多层之间转换格式。
- 枚举、时间、金额、权限在前后端都有含义。
- 修改一个字段可能影响多个消费者。

## 数据流检查

- 前端表单字段如何转换为 RequestVO。
- Controller 如何校验 RequestVO 并转换为 DTO。
- Service 如何基于 DTO 执行业务规则和事务。
- Mapper 如何查询或写入数据库。
- DTO 如何转换为 ResponseVO 返回给前端展示。

## 契约检查

- 字段是否可空。
- 枚举 code 是否稳定。
- 时间单位和时区是否明确。
- 金额精度是否一致。
- 错误码是否被前端消费。

## 风险检查

- 是否存在权限只在前端控制的问题。
- 是否存在前端传排序字段导致 SQL 注入的问题。
- 是否存在 Entity 直接暴露给前端的问题。
- 是否存在缓存写后不一致的问题。
