# 数据库结构变更指南

## 变更前

- [ ] 搜索字段、表名、枚举值的所有引用。
- [ ] 判断是否为兼容变更。
- [ ] 评估数据量、锁表风险和回滚方案。
- [ ] 确认索引、约束、默认值和历史数据处理策略。

## 迁移脚本

- 使用 Flyway 或 Liquibase 管理脚本。
- 脚本命名必须可排序且语义清晰。
- 禁止手工直接修改生产库而不同步脚本。
- 大表变更必须评估在线 DDL 方案。

## 代码同步

- 更新 Entity、Mapper、DTO、VO、转换器。
- 更新 API 文档和前端类型。
- 更新测试数据和集成测试。
- 修改字段语义时同步所有读写路径。

## 上线策略

- 兼容性变更优先采用 expand-contract 模式。
- 先新增字段并双写，再迁移读路径，最后删除旧字段。
- 发布前确认迁移脚本已在测试环境执行。
