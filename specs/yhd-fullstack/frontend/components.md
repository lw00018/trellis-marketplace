# 组件规范

## 组件职责

- 展示组件只接收 props，不直接请求后端。
- 容器组件负责组合状态、请求和业务交互。
- 表单组件必须明确受控状态、校验状态和提交状态。
- 复杂业务组件应放在 feature 内，不放入全局 components。

## 命名

- 组件使用 UpperCamelCase。
- 业务组件名称包含领域语义，例如 `UserRoleEditor`。
- 通用组件名称表达 UI 语义，例如 `ConfirmDialog`。

## 可维护性

- 不在组件中硬编码后端错误码处理。
- 不在组件中拼接 API URL。
- 不把大型页面写成一个巨型组件。
- 组件边界应以复用和职责清晰为准，而不是机械拆分。
