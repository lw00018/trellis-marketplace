# 前端身份认证规范

## 登录态管理

- 登录态必须由统一认证模块管理。
- 页面不得各自读取、解析或刷新 Token。
- 认证初始化期间必须有明确加载态，避免闪烁到未授权页面。
- 登出必须清理认证状态、缓存和敏感页面数据。

## Token 存储

- 优先使用 HttpOnly Cookie 承载敏感 Token。
- 如果必须使用浏览器存储，必须评估 XSS 风险并缩短有效期。
- 刷新 Token 逻辑必须集中处理，避免并发刷新风暴。

## 权限展示

- 前端权限只用于改善体验，不作为安全边界。
- 按钮和菜单展示必须与服务端权限模型对齐。
- 服务端返回 403 时必须正确处理，而不是假设前端已拦截。
